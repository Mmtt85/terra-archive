# 인프라 플래너 규칙서

테라 아카이브 인프라 플래너(`app/planner.tsx` + `scripts/build-infra.py`)가 따르는 모든 도메인
규칙의 정본. 새 오퍼레이터가 나오거나 편성이 이상할 때 이 문서를 기준으로 파서·모델을 갱신한다.
데이터 갱신 절차 자체는 `scripts/README.md` 참고.

## 1. 기지 구성과 교대 정책

- 레이아웃은 **243 고정**: 무역소 2 · 제조소 4 · 발전소 3 + 제어 센터, 숙소 4, 응접실·가공소·사무실·훈련실.
- 제조소는 **순금 2 + 작전기록 2** 분할. KR 데이터 표기는 '금괴'가 아니라 **'순금'/'귀금속류'**.
- **A조가 풀파워 주력**이고 모든 시너지 세트는 A조에 집중한다. B조는 A조 컨디션 소진 시 투입되는
  회복 교대(12시간 2조). 3조 이상 교대는 쓰지 않는다.
- **순금이 병목**: 무역소 효율이 오르면 순금 소비가 생산을 앞지르므로, 가장 강한 생산 팀을 순금
  2방에 먼저 앉히고 남는 효율을 작전기록으로 돌린다.
- **훈련실은 비워 둔다** (실제 스킬 특화 훈련 용도). **가공소는 니엔 고정**이 정배.
- **숙소는 항상 풀 인원(방당 5명) 기준**으로 계산하며, 조 전환과 무관하게 고정. 숙소 시간당 회복
  스킬로 배치를 최적화하지는 않는다 — 숙소 슬롯은 휴식 + 토큰 생성원 고정용.
- **응접실은 A/B 전환과 별개의 상시 편성**.
- **정예화 단계 기본값 = 최대 정예화** (4성+ = 2정, 3성 = 1정, 1~2성은 승급 없음). 정예화
  해금 인프라 스킬(unlock이 `"정예화 1"`/`"정예화 2"`)이 있는 오퍼만 보유 오퍼 설정·방 상세에서
  **노정예/1정/2정**으로 낮출 수 있고, 낮추면 해당 단계 미만 해금 스킬이 스코어링·자동편성·방
  상세 표시 전부에서 제외된다(`withElite()`, `app/planner.tsx`). 성급별 일괄 버튼 제공
  (3성 이하는 2정이 없으므로 보유/해제만).

## 2. 스킬 적용 규칙 (데이터 구조가 정답)

- 강화/신규 구분은 `building_data`의 **buffChar 슬롯 구조**로 판정한다:
  - 같은 슬롯 안의 buffData 항목들 = **강화 단계** → 최종 단계만 적용
    (예: 피구리노 맞춤 재단 20% → 자유자재 25%, 합산 아님).
  - 서로 다른 슬롯 = **별개 스킬, 동시 적용** (예: 샤마르 재봉 α + 속삭임).
  - 이름 휴리스틱(α/β 접미사, 설명문 유사도)으로 추측하지 말 것.
- 수치 표기는 `+30%`와 `30% 상승` 두 형태. 정규식 필러가 숫자를 먹지 않도록 `[^%\d]` 계열을 쓴다
  ("50% 상승"이 "0% 상승"으로 잡히는 백트래킹 버그 전례 있음).
- 성장형 상한: "이후 시간당 2% 상승, **최대 30%까지**" → 상한값을 채택 (이네스).
  발전소도 동일 — "드론 상한 10당 +1% (최대 +25%)" → **+25% 고정** (이격그레이 점검 매뉴얼, 사용자 확정).

## 3. 시설 간 포인트 토큰 시스템

토큰 = 시설을 넘나드는 포인트. 생성자(+N)와 소비자(N점/개당 +효과)로 구성.

| 토큰 | 생성 | 소비 | 비고 |
|---|---|---|---|
| 속세의 화식 | 시·링(제어센터 +15), 총웨(쉐이 1명당 +5, 최대 5명 — **실제 배치 수로 재계산**), 우요우(숙소 인원당 +1), 멀베리(모집당) | 슈(제조 3점당 생산력+1%), 우요우(오더 효율 1점당+1%), 위(훈련), 윈드차임스(컨디션) | 쉐이 세트의 축 |
| 주술 결정 | (직접 생성 없음) | 지에윈(제조 1점당 생산력+1~2%) | **화식 5점→1점 전환** (지에윈 자신) |
| 감지 정보 | 시·링(조건부), 에벤홀츠(숙소 인원당), 아이리스 꿈나라 스택, 체르니 소절 스택(숙소 레벨당, Lv5=5) | (직접 소비 소수) | 공명의 원료 |
| 무성의 공명 | 비르투오사(숙소 인원당 +1) | 에벤홀츠(무역 2점당 오더 효율+1%) | **에벤홀츠가 감지 정보→공명 1:1 전환** |
| 생각의 사슬 | 로즈몬티스 | 로즈몬티스(제조) | 단독 자기완결형 |
| 마물 요리 | 센시(숙소 **레벨당 1개** = 5개, 숙소 고정 필요) | 마르실(제조 개당+1%), 라이오스(응접실 개당+2%), 칠책(무역) | 라이오스 파티(던전밥) |
| 정보 저장 | Ash(제어센터 내 레인보우 팀 1명당 +1) | Blitz·Frost·Iana | **레인보우 팀 전용 폐쇄 시스템 — 기지 패키지에서 제외** |

- **폐쇄 팀 규칙**: 참가자 전원이 단일 진영이고 2명 이상이면 기지 전체 패키지에서 제외한다.
  예외: 생성원이 전부 **숙소**에 있으면 허용(센시 마물 요리 — 경쟁 슬롯을 침범하지 않으므로).
- 숙소 기반 생성원(아이리스·체르니·비르투오사·센시)은 **숙소에 고정**되고 조 전환에 영향받지 않는다.
- 시드는 토큰 기대 가치 **20% 이상인 핵심 소비자만** 예약 배치한다. 약한 소비자(마르실 +5%급)는
  일반 경쟁 — 토큰 값은 점수에 자동 반영되므로 자리 낭비를 막는다.
- 시드된 오퍼는 해당 방에 **예약**된다(먼저 배정되는 방이 그리디로 채가지 못함).

## 4. 특수 역할 (무역소·제조소)

- **효율 오버라이드(샤마르 속삭임)**: 같은 무역소 타인의 오더 효율을 전부 0으로 만들고 인당 +45%.
  → 방 효율 = 45 × 인원. 효율 요원과 섞지 말 것.
- **고품질 확률(재봉/수공예/감정 계열)**: 샤마르·카프카·디아만테·바이비크 — 전부 동급 대체 가능
  (α 소폭=10, β=15 등가).
- **품질 수익(테킬라 투자β)**: "순금 오더 납품 3초과" = 고품질 귀금속 오더. 확률 요원 수에 비례해
  증폭 → **샤마르+테킬라+확률 요원 1명**이 완성형.
- **위약 수익(프로바이조)**: 계약법이 납품 4미만을 위약 오더로 만들고 배상 +2. 고품질 확률과는
  **반시너지**(저품질 오더가 필요), 대신 **방 처리량(효율)에 비례해 증폭** →
  **우요우+에벤홀츠+프로바이조**가 완성형.
- **자동화(위디·유넥티스·윈드플릿·패신저)**: 방 내 오퍼 제공 생산력을 0으로 만들고
  **발전소 1기당** +15%/+10%/+5%/+5%. 단 "(시설 수량에 따라 제공하는 생산력 미포함)" —
  **시설 기반 생산력은 생존**: 퓨어스트림(무역소당 귀금속 +20%), 쏜즈 로드스타(무역소당 +3%) →
  **위디+유넥티스+퓨어스트림** 순금방 성립.
- **자동화 · 인원수 스케일(스네구로치카)**: 같은 제로아웃 메커니즘이지만 발전소가 아니라
  **그 제조소에 실제 배치된 인원수 1명당** +10% — `kind: "automation_crew"`
  (`app/planner.tsx`의 `breakdown()`에서 `teamSize`로 스케일, `scripts/build-infra.py`
  파서는 "생산력이 전부 0이 되고 … 제조소 내의 오퍼레이터 1명당" 문구로 감지).
  일반 진영-카운트 정규식이 "제조소 **내의** 오퍼레이터 1명당"의 "내의"를 진영명으로 오인식하는
  버그가 있었음 — `내의`/`안의`/`내`는 진영 캡처 제외 목록에 있다.
- **제로아웃 오퍼 추천 원칙**: 방 내 모든 인원의 생산력을 0으로 만드는 오퍼(위 두 항목)를
  일반 +N% 생산력 오퍼와 같은 방에 추천하면 안 된다 — 그 +N%는 전부 0이 된다. 자동편성은
  실제 방 점수(제로아웃 반영)로만 비교하므로 이미 안전하지만, 새 오퍼 데이터를 다룰 때는
  "생산력이 전부 0이 되고" 문구가 있는데 `kind`가 `automation`/`automation_crew`/`override`가
  아니면(예: `output`으로 잘못 분류) 파서 버그다 — 제로아웃 오퍼를 쓰는 편성 자체가 예외적인
  케이스라는 것을 전제로 점검한다.
- **그레이 더 라이트닝베어러**: "다른 발전소에 배치된 작업 플랫폼이 없을 경우 발전소 +1개 간주".
  **작업 플랫폼 = 1성 로봇**이므로 로봇을 발전소에 안 넣으면 조건 충족 → 발전소에 배치하면
  발전소 4기로 계산되어 자동화 방이 140%가 된다.

## 5. 조건부 스킬 판정

- **진영 동반 게이트**: "X 오퍼레이터와 함께 배치 시" → 같은 방에 그 진영 오퍼(본인 제외)가 실제로
  있어야 발동 (호시구마 더 브리처 동료애: 용문근위국).
- **진영 카운트**: "X 오퍼레이터 1명당 +v% (최대 c%)" → 실제 배치 인원 기준, 상한 적용
  (바르카리스: 미노스). 범위는 "제어 센터 내"면 방, 아니면 기지 전체.
- **동방 스킬 카운트**: "제조소 내 금속 공예류 스킬 1개당 +5%" → 팀원의 활성 스킬 이름 매칭
  (브라이오피타).
- **시설 카운트**: "각각의 무역소가 … +3%" → 무역소 2 / 발전소 3(또는 4) / 제조소 4 배수.
- **동료 카운트**: "자신을 제외한 작업 중인 오퍼레이터 1명당 +10%" (키치세이).
- **단독 조건**: "자신만 업무 중일 때" → 혼자일 때만 (아르모니). **단서 공유 조건**: "단서 공유
  상태에서" → 액면가로 인정.
- **레벨 성장형은 만렙 기지 기준 상한 채택** (이네스·이격그레이 원칙의 확장, 사용자 확인):
  - "응접실 레벨 1레벨당 추가 +5%, 최대 40% 제공" → 응접실 Lv3 = **상한값 채택** (비질 40, 미틈 30).
  - "**모든 숙소의 레벨 1당** +N%" → 숙소 4 × Lv5 = **×20** (아르케토 40, 틴맨 45,
    나란투야 20(순금), 필라에 10). 파서는 이 절을 떼고 기본치를 파싱한 뒤 ×20을 더한다.
  - "정예 오퍼레이터가 배치된 시설 1개당 +2% (최대 10개)" → 풀 기지는 시설 10개를 쉽게
    채우므로 상한 채택 (만트라 45).
- **기반시설 어디든 존재 조건 (`basePartners`)**: "울피아누스가 **기반시설에** 있으면 추가로
  +10%" (언더플로우) — 같은 방 동반(partners)이 아니라 **기지 전체(숙소 포함) 존재** 조건.
  partners로 오인하면 기본 +30%까지 통째로 게이트되는 버그가 있었다. 플래너는 조별 배치
  전원(`presentIds`)으로 판정 — 울피아누스를 숙소에 하나 고정해 두면 항상 발동한다.
  사용자 확인 정배: **B조 무역소 비질(40)+아르케토(40)+언더플로우(30→40) + 숙소 울피아누스 고정**.

## 6. 제어 센터

- 오라 우선순위(사용자 확정): **제조소 생산력 > 무역소 오더 효율 > 인맥 레퍼런스 > 단서 수집**.
  가중치: mfg ×10, trade ×2, hire ×0.6, clue ×0.2.
- "동종 효과 중 가장 높은 수치만 적용" — 같은 종류 오라는 팀 내 최대값 하나만.
- 쉐이 3인(링·시·총웨)이 화식 생성으로 고정, 남는 2자리를 오라 요원으로 채움.

## 7. 대체 추천

- **슬롯별 교체 시뮬레이션**: 그 자리에 후보를 넣었을 때의 방 점수로 순위. 동점이면 저성급 우선
  (육성 저렴 — 그라벨 원칙).
- **시너지 코어는 대체 불가**: 활성 토큰의 생성자/소비자, 오버라이드·수익 역할, 인원 카운터 대상
  (쉐이 진영) → "대체 불가 · 시너지 코어" 표시. 디아만테처럼 역할 호환 자리만 대체 후보 제공.

## 9. 신규 오퍼레이터 추가 시 점검 목록

1. `python3 scripts/build-infra.py <gamedata>` 재실행 후 신규 오퍼의 파싱 결과 확인
   (kind / value / product / tokenGen / tokenUse / 조건 필드).
2. 새 토큰 시스템이면 `TOKENS` 목록에 추가하고 생성·소비·전환 문구가 잡히는지 확인.
3. "…당", "…와 함께", "전부 0이 되고", "간주" 같은 새 조건 문구는 이 문서에 규칙을 추가하고
   파서·플래너에 반영.
4. 플래너 로직 검증은 `app/planner.tsx`의 컴포넌트 이전 부분을 esbuild로 번들해 노드에서
   `optimize(ops)`를 직접 실행 (과거 검증 스크립트 참고).
5. 확정된 새 규칙은 이 문서와 도움말 모달(`planner.tsx`의 HelpModal)에 함께 갱신.
