# 테라 아카이브 유지보수 오케스트레이터

무인 GitHub Actions가 자동으로 못 하는 일(사람 판단이 필요한 항목)을 **한 번에 감지해
필요한 하위 스킬만 골라 실행**한다. 메일이 어떤 스킬을 지목하든, 독타는 이 스킬 하나만
돌리면 된다. 아래 순서대로 감지하고, 걸린 항목마다 해당 하위 스킬을 **Skill 도구로 호출**해
그 절차를 그대로 따른다.

## 0. 최신 데이터 반영 확인 (선택)

무인 결정론 레인이 이미 돌았다면 `app/data/*.json`은 최신이다. 확신이 없으면 로컬에서
한 번 리프레시해 최신 상태에서 감지한다 (네트워크·수 분 소요):
```bash
bash scripts/ci-refresh.sh
```

## 1. 감지 — 무엇이 밀렸는지 스캔

```bash
node scripts/ci-report.mjs kr          # → .ci/report.md 의 "🖐 사람 손 필요" 섹션
node scripts/ci-detect-content.mjs     # → 요약 없는 신규 이벤트 (.ci/content-tasks.json)
```
`.ci/report.md`와 `.ci/content-tasks.json`을 읽어 아래 5개 범주 중 걸린 걸 추린다.

## 2. 범주별 하위 스킬 실행 (걸린 것만)

| 감지 신호 | 실행할 하위 스킬 |
|---|---|
| `content-tasks.json`에 이벤트 있음 (요약 없는 신규 이벤트) | **story-summary** |
| report에 "테라 연대기 미등록" | **chronicle-register** |
| report에 "신규 오퍼 인프라 시너지 검토" | **planner-synergy-review** |
| report/경고에 "CN 신규 텍스트 번역"(未/译/미번역) | **cn-translation-fill** |
| 새 통합전략(록라) 토픽 등장 | **rogue-guide** |

각 범주에서 걸린 게 있으면 해당 스킬을 `Skill` 도구로 호출해 절차를 수행한다.
여러 범주가 걸리면 **story-summary → chronicle-register 순서**를 지킨다(요약이 있어야
연대기의 테라력·아크 판단이 쉬움). 나머지는 순서 무관.

## 3. 통합 마무리

- 하위 스킬들이 각각 데이터를 고쳤으면, **마지막에 한 번** 빌드 확인 → 커밋 → push.
  (하위 스킬이 개별 커밋하지 않았다면 여기서 모아서 커밋.)
- KR 데이터를 재생성했으면 `build-i18n.py`로 EN/JA도 함께 재생성됐는지 확인 (CLAUDE.md 규칙).
- 배포는 자동 실행 금지 — 사용자가 직접 (`docs/AUTOMATION.md`).
- **처리 결과 요약 보고**: 어떤 범주를 무엇으로 처리했는지, 사람 확인이 더 필요한 게 남았는지.

## 아무것도 안 걸렸으면

감지에서 5개 범주 모두 비었으면 "밀린 수작업 없음"이라고 보고하고 끝낸다 — 억지로 만들지 말 것.
