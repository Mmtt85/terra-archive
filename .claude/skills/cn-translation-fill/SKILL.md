# 미실장(CN) 신규 텍스트 번역

미래시(중국 선행) 오퍼·재료의 상세 텍스트(특성·재능·스킬·잠재·모듈·기반시설·모집사유,
재료 이름·설명)는 클뜯에 KR 번역이 아직 없어 중국어 원문 그대로다. `scripts/cn-translations.json`
(`{cn원문: {ko, en, ja}}`)에 **비공식 AI 번역**을 채우면 파이프라인이 이를 오버레이한다.
상세는 [scripts/README.md](../../../scripts/README.md) §2 / §5.5.

## 절차

1. **미번역 원문 수집** — 파이프라인을 돌리면 미번역 중국어 원문을 경고로 출력한다:
   ```bash
   bash scripts/ci-refresh.sh 2>&1 | grep -Ei "미번역|未|译" | sort -u
   ```
   또는 무인 리포트 이메일의 "파이프라인 경고" 블록에 찍힌 원문을 쓴다.

2. **cn-translations.json에 번역 추가** — 각 원문을 키로, ko/en/ja를 채운다:
   ```json
   "<중국어 원문 그대로>": { "ko": "<한국어>", "en": "<English>", "ja": "<日本語>" }
   ```
   - 원문 키는 **글자 하나도 바꾸지 말 것** (공백·문장부호 포함 정확히 일치해야 매칭됨).
   - 명일방주 공식 용어·표기를 따른다(진영명·재료명 등). 확신 없으면 기존 항목 스타일 참고.
   - 게임 도메인 용어는 이미 채워진 302개 항목의 번역 톤을 맞춘다.

3. **재생성 + 확인** — 번역이 반영되는지, 남은 미번역이 없는지:
   ```bash
   bash scripts/ci-refresh.sh 2>&1 | grep -Ei "미번역|未|译" | sort -u   # 비어야 정상
   ```
   (regen-operators / build-costs / build-i18n가 이 파일을 읽어 operators.json·costs.json·
   EN/JA 데이터에 반영한다. build-i18n는 ko→en/ja 대응도 이 파일에서 수확.)

4. 빌드 확인 → 커밋 → push. 배포는 사용자가 직접.

## 주의

- UI에는 이미 '비공식 AI 번역' 고지가 뜬다(i18n.tsx) — 정식 KR 번역이 나오면 클뜯이
  자동 대체하므로 이 파일 항목은 그대로 둬도 무방(원문이 안 잡히면 미사용).
- 미실장 오퍼는 인프라 플래너·공채에서 KR 데이터 기반이라 자동 제외되고, '미래시 데이터 포함'
  토글로만 노출된다 (INFRA-RULES §9). 번역은 표시용일 뿐 계산에 영향 없음.
